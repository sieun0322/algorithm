# Step 6: Test Cases (Code Analysis)

## [Code] TC11: 연쇄 합침
- **Input**: `[[1,2,3,5],...]`, "right"
- **Expected**: `[[0,0,3,8],...]`
- **Reason**: 1+2=3, 3+5=8 (두 번 합침)

## [Code] TC12: 합침 후 이동
- **Input**: `[[1,1,0,2],...]`, "right"
- **Expected**: `[[0,0,2,3],...]`
- **Reason**: 1+1=2 후, 새 2가 기존 2와 만남 → 2+2=4 (not fib) 안 합침. 잠깐, 1+1=2 후 [2,2] → [0,0,2,2]? 아니, 2+2=4 안됨. 그런데 원래 2가 있고...
- **재분석**: [1,1,0,2] compact → [1,1,2], 1+1=2, [2,2], 2+2=4 X → [0,0,2,2]
- **수정**: `[[0,0,2,2],...]`

## [Code] TC13: 같은 1 네 개
- **Input**: `[[1,1,1,1],...]`, "right"
- **Expected**: `[[0,0,2,2],...]`
- **Reason**: (1+1)=2, (1+1)=2

## [Code] TC14: 큰 피보나치 합침
- **Input**: `[[610,987,0,0],...]`, "right"
- **Expected**: `[[0,0,0,1597],...]`
- **Reason**: 610+987=1597
