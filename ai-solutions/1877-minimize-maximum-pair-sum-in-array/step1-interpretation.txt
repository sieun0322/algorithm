## 1. Problem Restatement

짝수 개의 숫자가 담긴 배열이 주어진다. 이 숫자들을 두 개씩 짝지어 쌍을 만들어야 한다. 각 쌍의 합 중에서 가장 큰 값을 "최대 쌍 합"이라고 한다. 목표는 짝짓는 방법을 잘 선택해서 이 "최대 쌍 합"을 가능한 한 작게 만드는 것이다.

## 2. Input Format

- `nums`: 정수 배열
- 배열 길이는 항상 짝수

## 3. Output Format

- 단일 정수: 최적으로 짝지었을 때의 최소화된 최대 쌍 합

## 4. Constraints

| 제약 | 값 |
|------|-----|
| 배열 길이 n | 2 ≤ n ≤ 10^5 |
| n의 특성 | 항상 짝수 |
| 각 원소 값 | 1 ≤ nums[i] ≤ 10^5 |

## 5. Pitfalls

- **모든 원소를 사용해야 함**: 일부만 선택하는 것이 아니라 전체를 짝지어야 함
- **중복 원소 존재 가능**: 같은 값이 여러 번 나올 수 있음 (예: [3,3])
- **짝짓기 조합의 수**: n/2개의 쌍을 만드는 경우의 수가 매우 많음 (brute force 불가)
- **최대 쌍 합의 최소화**: 단순히 전체 합을 최소화하는 것이 아님
